### Бизнес-постановка задачи
**Цель**
  * создание MailLens, сервиса интеллектуальной категоризации почтового трафика.

  * Сервис должен классифицировать письма по произвольно заданным пользователем категориям, определять близость письма к каждой категории и корректно обрабатывать случаи, когда письмо не подходит ни под одну категорию («Не определена»).

  * Решение предназначено для структурирования корпоративного почтового потока: от бизнес-корреспонденции до потенциально вредоносных писем.


**Этапы реализации**
1. **Подготовка и обработка данных**
    * Парсинг писем .eml/.msg:
      * извлечение темы, тела, HTML, plain-text частей
      * извлечение вложений (текстовых) и ссылок

    * Ручная предобработка текстов:
      * удаление html-тегов, служебных блоков, мусора
      * нормализация пробелов и переносов строк
      * объединение частей письма в единый текст (subject + body + attachments summary)
    * Формирование структуры данных:
      * единый словарь вида {id, subject, body, attachments_text, links, full_text}

2. **Генерация эмбеддингов**
    * Подготовка текстов:
      * формирование эмбеддинга письма
      * формирование эмбеддингов названий категорий
    * Получение векторных представлений:
      * создание матрицы категории × размерность эмбеддинга
      * хранение и кеширование эмбеддингов категорий для ускорения работы.
      * введение порога threshold для максимального сходства;

3. **Построение классификатора**
    * вычисление косинусного сходства между email_vector и category_vectors
    * введение порога threshold для максимального сходства
    * проверка устойчивости к шуму (спам, фишинг, короткие письма).


4. **Оценка качества**
    * Вычисление метрик:
      * Accuracy (для категорий, кроме «Не определена»)
      * Precision/Recall/F1 по категориям
      * отчёт по случаям, попавшим в «Не определена»

5. **Разработка backend-сервиса**
    * загрузка файла .eml/.msg;
    * возврат результатов
    * логирование
    * экспорт results.csv

6. **Разработка веб-интерфейса**
    * Создание формы загрузки письма
    * Отображение результата
    * возможность скачать CSV

7. **Тестирование**
    * Внедрение на этапе разработки тестов assert
    * Проверки на ошибки в работе UI и модели

8. **Документация и сопровождение**
    * Подготовка README
    * Подготовка requirements файлов
    * Презентация

## Установка
В дирректории docker необходимо выполнить следующие команды:
```python
docker compose build
docker compose up -d
```
После чего приложение будет доступно по localhost в браузере.
